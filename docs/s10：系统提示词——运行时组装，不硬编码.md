# s10：系统提示词——运行时组装，不硬编码

s01 到 s09 逐步加入了工具、权限、Hook、Todo、子 Agent、Skill、Context Compact 和 Memory。能力越多，system prompt 越不能继续写成一整段硬编码字符串。s10 的目标是把 system prompt 变成运行时组装的 section 系统。

当前项目中，主 Agent 和子 Agent 都通过同一个 `system_prompt_builder.py` 获取 system prompt：

```text
agent_loop.py
-> build_main_system_prompt(...)

subagents/task_subagent.py
-> build_subagent_system_prompt(...)
```

## 文件结构

```text
configs/
└── prompt_config.yml
state/
└── prompt_state.py
prompts/
├── prompt_sections.py
├── sections.py
├── memory_prompts.py
├── system_prompts.py
└── subagent_prompts.py
managers/
└── system_prompt_builder.py
```

职责：

- `configs/prompt_config.yml`：控制 prompt 缓存、section 注入开关、项目指令文件、静态/动态边界。
- `state/prompt_state.py`：定义 prompt section、prompt context、构建结果。
- `prompts/prompt_sections.py`：定义主 Agent / 子 Agent 的 section 顺序和静态 prompt 片段。
- `prompts/sections.py`：兼容导出层。
- `managers/system_prompt_builder.py`：按运行时状态组装最终 system prompt。

## 配置

当前 `configs/prompt_config.yml`：

```yaml
ENABLE_PROMPT_CACHE: true
INJECT_TOOL_SUMMARY: true
INJECT_SKILL_CATALOG: true
INJECT_MEMORY_INDEX: true
INJECT_PROJECT_INSTRUCTIONS: true
PROJECT_INSTRUCTION_FILES:
  - "CLAUDE.md"
  - ".claude/CLAUDE.md"
  - "AGENTS.md"
MAX_PROJECT_INSTRUCTION_CHARS: 12000
INJECT_RUNTIME_CONTEXT: true
STATIC_DYNAMIC_BOUNDARY: "<SYSTEM_PROMPT_DYNAMIC_BOUNDARY>"
DEBUG_PROMPT_SECTIONS: false
```

说明：

- `ENABLE_PROMPT_CACHE`：启用字符串级缓存。
- `INJECT_TOOL_SUMMARY`：把工具摘要写进 system prompt。
- `INJECT_SKILL_CATALOG`：注入 Skill 目录，但不注入完整 `SKILL.md`。
- `INJECT_MEMORY_INDEX`：注入 Memory 规则和 `MEMORY.md` 索引，但不注入全部记忆正文。
- `INJECT_PROJECT_INSTRUCTIONS`：读取 `CLAUDE.md` / `.claude/CLAUDE.md` / `AGENTS.md`。
- `STATIC_DYNAMIC_BOUNDARY`：静态 section 和动态 section 的分隔符。

## 状态对象

`state/prompt_state.py` 定义三个 dataclass：

```python
PromptSection(
    name: str,
    content: str,
    dynamic: bool = False,
)
```

表示一个 system prompt section。

```python
SystemPromptContext(
    workdir: str,
    agent_type: str = "main",
    enabled_tools: List[str] = [],
    skill_catalog: str = "",
    memory_index: str = "",
    project_instructions: str = "",
    runtime_context: Dict[str, Any] = {},
)
```

表示构建 system prompt 需要的运行时状态。

```python
PromptBuildResult(
    text: str,
    section_names: List[str],
    cache_hit: bool = False,
)
```

表示一次组装结果。

## 主 Agent 构建流程

主 Agent 在 `agent_loop.py` 中调用：

```python
build_main_system_prompt(str(WORKDIR))
```

调用链：

```text
build_main_system_prompt(workdir)
-> _tool_names()
-> SystemPromptContext(agent_type="main")
-> get_system_prompt(context)
-> _cache_key(context)
-> 缓存命中：直接返回上次字符串
-> 缓存未命中：assemble_system_prompt(context)
-> _build_sections(context)
-> _section_builders(context)
-> _assemble_text(sections)
-> 返回最终 system prompt
```

主 Agent section 顺序来自 `MAIN_SECTION_ORDER`：

```python
[
    "identity",
    "workspace",
    "tool_rules",
    "tool_summary",
    "planning_rules",
    "permission_rules",
    "skills_index",
    "memory_rules_and_index",
    "project_instructions",
    "runtime_context",
    "response_style",
]
```

对应构建函数：

```text
identity
-> MAIN_IDENTITY_SECTION

workspace
-> _workspace_section(context)

tool_rules
-> TOOL_RULES_SECTION

tool_summary
-> _tool_summary(context)

planning_rules
-> PLANNING_RULES_SECTION

permission_rules
-> PERMISSION_RULES_SECTION

skills_index
-> _skill_catalog_prompt()

memory_rules_and_index
-> _memory_index_prompt("main")

project_instructions
-> _project_instructions()

runtime_context
-> _runtime_context(context)

response_style
-> RESPONSE_STYLE_SECTION
```

## 子 Agent 构建流程

子 Agent 在 `subagents/task_subagent.py` 中调用：

```python
build_subagent_system_prompt(workdir)
```

调用链：

```text
build_subagent_system_prompt(workdir)
-> 读取 SUBAGENT_AVAILABLE_TOOLS
-> _tool_names(SUBAGENT_AVAILABLE_TOOLS)
-> SystemPromptContext(agent_type="subagent")
-> get_system_prompt(context)
-> assemble_system_prompt(context)
-> _build_sections(context)
-> _assemble_text(sections)
```

子 Agent section 顺序来自 `SUBAGENT_SECTION_ORDER`：

```python
[
    "subagent_identity",
    "workspace",
    "subagent_context_rules",
    "tool_rules",
    "tool_summary",
    "skills_index",
    "memory_rules_and_index",
    "project_instructions",
    "runtime_context",
    "subagent_result_rules",
]
```

子 Agent 的工具摘要只显示 `SUBAGENT_AVAILABLE_TOOLS` 中实际可用的工具。例如当前配置允许 `load_skill`，但不会显示主 Agent 的 `task`、`save_memory`、`forget_memory`。

子 Agent 使用 `MEMORY_READ_ONLY_RULES`，只读 Memory，不提示它调用写入 Memory 的工具。

## Section 构建

所有 section 构建函数集中在 `_section_builders(context)`：

```python
{
    "identity": lambda: MAIN_IDENTITY_SECTION,
    "subagent_identity": lambda: SUBAGENT_IDENTITY_SECTION,
    "workspace": lambda: _workspace_section(context),
    "subagent_context_rules": lambda: SUBAGENT_CONTEXT_RULES_SECTION,
    "tool_rules": lambda: TOOL_RULES_SECTION,
    "tool_summary": lambda: _tool_summary(context),
    "planning_rules": lambda: PLANNING_RULES_SECTION,
    "permission_rules": lambda: PERMISSION_RULES_SECTION,
    "skills_index": _skill_catalog_prompt,
    "memory_rules_and_index": lambda: _memory_index_prompt(context.agent_type),
    "project_instructions": _project_instructions,
    "runtime_context": lambda: _runtime_context(context),
    "response_style": lambda: RESPONSE_STYLE_SECTION,
    "subagent_result_rules": lambda: SUBAGENT_RESULT_RULES_SECTION,
}
```

`_build_sections(context)` 会：

1. 根据 `agent_type` 选择主 Agent 或子 Agent 的 section 顺序。
2. 按顺序调用 section builder。
3. 跳过空 section。
4. 根据 `STATIC_SECTIONS` 标记静态或动态。

## 静态和动态边界

静态 section 定义在 `STATIC_SECTIONS`：

```python
{
    "identity",
    "subagent_identity",
    "subagent_context_rules",
    "tool_rules",
    "planning_rules",
    "permission_rules",
    "response_style",
    "subagent_result_rules",
}
```

`_assemble_text(sections)` 会把 section 分为：

```text
静态 section
<SYSTEM_PROMPT_DYNAMIC_BOUNDARY>
动态 section
```

当前这只是字符串边界。OpenAI Chat Completions 仍然收到一条完整 system message。这个边界主要用于调试，也为未来接入 API prompt cache 留位置。

主 Agent 最终大致顺序：

```text
静态：
1. identity
2. tool_rules
3. planning_rules
4. permission_rules
5. response_style

边界：
<SYSTEM_PROMPT_DYNAMIC_BOUNDARY>

动态：
1. workspace
2. tool_summary
3. skills_index
4. memory_rules_and_index
5. project_instructions
6. runtime_context
```

子 Agent 最终大致顺序：

```text
静态：
1. subagent_identity
2. subagent_context_rules
3. tool_rules
4. subagent_result_rules

边界：
<SYSTEM_PROMPT_DYNAMIC_BOUNDARY>

动态：
1. workspace
2. tool_summary
3. skills_index
4. memory_rules_and_index
5. project_instructions
6. runtime_context
```

## 动态 section 来源

### 工具摘要

`_tool_summary(context)` 从 `BASE_TOOLS` 读取工具名和 description。

主 Agent 使用全部 `BASE_TOOLS`。

子 Agent 通过 `enabled_tools` 过滤，只显示 `SUBAGENT_AVAILABLE_TOOLS` 中的工具。

注意：完整工具 schema 不放进 system prompt，因为模型调用时已经通过 `tools=BASE_TOOLS` 或子 Agent tools 参数传入。

### Skill 目录

`_skill_catalog_prompt()` 调用：

```python
SKILL_REGISTRY.list_catalog()
```

只注入技能名和简短描述。完整技能正文仍需要调用 `load_skill(name)`。

### Memory 索引

`_memory_index_prompt(agent_type)` 调用：

```python
MEMORY_MANAGER.load_index_prompt()
```

它读取：

```text
.memory/private/MEMORY.md
.memory/team/MEMORY.md
```

主 Agent 使用 `MEMORY_SYSTEM_RULES`，可以调用 Memory 写入工具。

子 Agent 使用 `MEMORY_READ_ONLY_RULES`，只读 Memory。

注意：相关 Memory 正文不会进入 system prompt，而是由 `memory_before_model_call_hook` 在 `BeforeModelCall` 阶段作为 `[Memory context]` 注入 messages。

### 项目指令文件

`_project_instructions()` 按配置读取：

```text
CLAUDE.md
.claude/CLAUDE.md
AGENTS.md
```

读取时使用 `safe_path(...)`，限制在当前工作区内，并受 `MAX_PROJECT_INSTRUCTION_CHARS` 控制。

### Runtime context

`_runtime_context(context)` 注入：

- 当前日期时间。
- Agent 类型。
- `runtime_context` 中额外传入的字段。

## 缓存机制

`get_system_prompt(context)` 使用全局单槽缓存：

```python
_PROMPT_CACHE_KEY = ""
_PROMPT_CACHE_TEXT = ""
```

流程：

```text
1. _cache_key(context) 生成稳定 key。
2. 如果 key 与上次一致，返回 _PROMPT_CACHE_TEXT。
3. 如果 key 不一致，重新 assemble。
4. 写入新的 key 和 text。
```

`_cache_key(context)` 使用 `json.dumps(..., sort_keys=True)`，不使用 Python `hash()`。

cache key 包含：

- workdir
- agent_type
- enabled_tools
- runtime_context
- skill catalog
- memory index
- project instructions
- prompt_config

这是字符串级缓存，不是 API prompt cache。主 Agent 和子 Agent 共用一个缓存槽，但 key 中包含 `agent_type` 和 `enabled_tools`，不会串内容；只是主/子切换时会互相覆盖缓存。

## 和其他模块的关系

### 和 agent_loop

`agent_loop.py` 只调用：

```python
build_main_system_prompt(str(WORKDIR))
```

它不关心 Skill、Memory、项目指令、工具摘要如何拼接。

### 和子 Agent

`task_subagent.py` 只调用：

```python
build_subagent_system_prompt(workdir)
```

子 Agent 的 prompt profile 更强调上下文隔离、只完成子任务、只读 Memory。

### 和 Skill Loading

system prompt 只注入 Skill 目录。完整技能说明仍通过 `load_skill(name)` 加载。

### 和 Memory

system prompt 只注入 Memory 索引和规则。相关记忆正文仍由 s09 的 `BeforeModelCall` Hook 注入 messages。

### 和 Context Compact

system prompt 不进入 `state.messages`，所以 s08 主要压缩的是对话历史和工具结果。Memory context 作为 user message 注入后，会参与 compact。

## 如何测试

### 查看主 Agent prompt

```powershell
python -c "from managers.system_prompt_builder import build_main_system_prompt; print(build_main_system_prompt('.'))"
```

应看到：

- 主 Agent 身份。
- 工具规则。
- 工具摘要。
- Skill 目录。
- Memory 索引。
- `<SYSTEM_PROMPT_DYNAMIC_BOUNDARY>`。
- runtime context。

### 查看子 Agent prompt

```powershell
python -c "from managers.system_prompt_builder import build_subagent_system_prompt; print(build_subagent_system_prompt('.'))"
```

应看到：

- 子 Agent 身份。
- 子 Agent 规则。
- 子 Agent 可用工具摘要。
- 只读 Memory 规则。
- 不应看到 `save_memory`、`forget_memory`、`task` 工具摘要。

### 检查 section 顺序

```powershell
python -c "from managers.system_prompt_builder import assemble_system_prompt; from state.prompt_state import SystemPromptContext; result=assemble_system_prompt(SystemPromptContext(workdir='.', agent_type='main', enabled_tools=['bash'])); print(result.section_names)"
```

应输出主 Agent section 名称列表。

## 验收标准

- 主 Agent 和子 Agent 都通过 `system_prompt_builder.py` 获取 system prompt。
- `agent_loop.py` 不包含大段 prompt 拼接逻辑。
- section 顺序由 `prompts/prompt_sections.py` 控制。
- 静态和动态 section 之间有 `<SYSTEM_PROMPT_DYNAMIC_BOUNDARY>`。
- 工具摘要随 `BASE_TOOLS` 或 `SUBAGENT_AVAILABLE_TOOLS` 自动变化。
- Skill 目录随 `skills/` 自动变化。
- Memory 索引随 `.memory/*/MEMORY.md` 自动变化。
- 项目存在 `CLAUDE.md` / `.claude/CLAUDE.md` / `AGENTS.md` 时能注入。
- 子 Agent prompt 不提示写 Memory，也不显示不可用工具。
- prompt context 不变时能复用缓存。
